声明

Go 的主要类型有，var、const、type 和 func，分别对应变量、常量、类型和函数的对象的声明。

一个 Go 语言编写的程序对应一个或多个以.go 为文件后缀名的源文件中。每个源文件以包的声明语句开始，说明该源文件是属于哪个包。 

const freezingF, boilingF = 32.0, 212.0

变量

var 变量名字 类型 = 表达式
其中“类型”或“= 表达式”两个部分可以省略其中的一个

var s string
var i, j , k, int
var b, f, s = true, 2, 3 // 类型能够推到得出, bool, float64, string

简短变量声明

在函数内部，有一种称为简短变量声明语句的形式可用于声明和初始化局部变量。它以“名字 := 表达式”形式声明变量，变量的类型根据表达式来自动推导。 因为简洁和灵活的特点，简短变量声明被广泛用于大部分的局部变量的声明和初化始。

var 形式的声明语句往往是用于需要显式指定变量类型地方，或者因为变量稍后会被重新赋值而初始值无关紧要的地方。 

i := 100
var boiling float64 = 100
var name []string
var err error
var p Point

i, j := 0, 1

注意， ：= 是一个变量声明语句，而 = 是一个变量赋值语句。 

i, j = j, i

简短变量声明语句也可以用函数的返回值来声明和初始化变量。 

f, err := os.Open(name)

下面是一个通过 flag 来构建一个命令行工具的例子 

package main

import (
    "flag"
    "fmt"
    "strings"
)

var n = flag.Bool("n", false, "omit trailing newline")
var sep = flag.String("s", " ", "separator")

func main() {
    flag.Parse()
    fmt.Print(strings.Join(flag.Args(), *sep))
    if !*n {
        fmt.Println()
    }
}

程序中的 sep 和 n 变量分别是指向对应命令行标志参数变量的指针，因此必须用*sep 和*n 形式的指针语法间接引用它们。

new

    fmt.Println(*p) // 0

包和文件

这里的包和其他语言中的库或者模块类似。每个包都对应一个独立的名字空间。 有一点要注意的是，一个程序的入口是 main package 下的 main 函数。

理解了包，我们其实就可以发布一个自己的包了。 下面以温度转换函数说明。

如果我们有一个包，其目录结构类似 $GOPATH/src/xxx/xxx，那我们在 import 这个包的时候就写 xxxx/xxxx 部分就可以了。 

    // src/ch2/tempconv
    package tempconv

    import "fmt"

    type Celsius float64
type Fahrenheit float64

const (
    AbsoluteZeroC Celsius = -273.15
    FreezingC     Celsius = 0
    BoilingC      Celsius = 100
)

另外还有一个文件 

// src/ch2/conv.go
package tempconv
func CToF(c Celsius) Fahrenheit { return Fahrenheit(c*9/5 + 32) }

// FToC converts a Fahrenheit temperature to Celsius.
func FToC(f Fahrenheit) Celsius { return Celsius((f - 32) * 5 / 9) }

下面我们在任何地方建一个 main.go 

    package main

import (
    "fmt"
    "ch2/tempconv"
)

func main() {
    fmt.Printf("%v\n", tempconv.AbsoluteZeroC)
}

上面有一点需要注意的是 package name 不一定和目录名相同。 也就是说，ch2/tempconv 我可以写成 ch2/haha。但是我们用的时候还是得用 tempconv 也就是 package name 而不能用 haha。 按照惯例，一个包的名字和包的导入路径的最后一个字段相同

关于 fmt 的技巧

通常 Printf 格式化字符串包含多个%参数时将会包含对应相同数量的额外操作数，但是%之后的[1]副词告诉 Printf 函数再次使用第一个操作数。第二，%后的#副词告诉 Printf 在用%o、%x 或%X 输出时生成 0、0x 或 0X 前缀。 

    o := 0666
fmt.Printf("%d %[1]o %#[1]o\n", o) // "438 666 0666"
x := int64(0xdeadbeef)
fmt.Printf("%d %[1]x %#[1]x %#[1]X\n", x)

438 666 0666
3735928559 deadbeef 0xdeadbeef 0XDEADBEEF

    ascii := 'a'
unicode := '国'
newline := '\n'
fmt.Printf("%d %[1]c %[1]q\n", ascii)   // "97 a 'a'"
fmt.Printf("%d %[1]c %[1]q\n", unicode) // "22269 国 '国'"
fmt.Printf("%d %[1]q\n", newline)       // "10 '\n'"

97 a 'a'
22269 国 '国'
10 '\n'

方法的声明

在函数声明时，在名字之前放一个变量，即是一个方法。 这个附加的参数会将函数附加到这种类型上。 

    package main

    import "math"

    type Point struct{x, y float64}

    func Distance(p, q Point) float64 {
    return math.Hypot(q.x - p.x, q.y - q.y)
}

    func (p Point) Distance(q Point) float64 {
    return math.Hypot(q.x - p.x, q.y - p.y)
}

    p := Point{1, 2}
q := Point{4, 6}
fmt.Println(Distance(p, q)) // "5", function call
    fmt.Println(p.Distance(q))  // "5", method call

接口

在 go 语言中，接口是非侵入式的设计。也就是说，你只需要去实现接口的方法，而不需要显式的继承这个接口。这种方式在我们定义一个新接口是可以不用改变已经存在的定义。

测试

同过 go test 来测试包目录内的以 _test.go 为结尾的测试文件。

测试函数必须导入 testing 包，名字必须以 Test 开头，可选的后缀名以大写字母开头 

func TestSin(t *test.T) {}

select

select 用法和 switch 有些类似。 

    timeout := make(chan bool, 1)
    ch := make(chan int)
    select {
    case <- ch:
            case <- timeout:
                    fmt.Println("timeout")
            default:
                    fmt.Println("default case is running")
}

如果没有 default, select 会一直等待某个 case 完成，否则会一直阻塞。

defer

defer 在 return 之前执行。 所谓之前是在 return 之后，REΤ 之后。 

    func f() (result int) {
    defer func() {
            result ++
    }()
            return 0
}

返回的结果为 1，而不是 0
