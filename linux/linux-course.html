  4. 创建新分区
    • 根分区 10 G
    • 交换分区 2 G
    • /boot 分区该分区用于存储内核和其他的启动信息。尽量将该分区设为磁盘驱动器的第一个物理分区。 100 M
    • /home 分区空间尽量大
    • /usr 分区 LFS 通常不需要
    • /opt 分区

    • /tmp 分区

      下面创建一个 root 分区和一个 swap 分区 

      fdisk /dev/sda

      输入 n， 然后接着输入 p, 然后是输入 1 为第一个分区。然后回车使用默认的 1，接着输入分区的大小，例如我输入的是 +16000M，你可以自己看着调整。

      下面创建 swap 分区。和上面差不多，输入 n, p, 2 然后回车使用默认的的大小。最后输入 t , 2, 82。

      最后，我们把我们的配置保存，使用命令 w。

      现在可以通过 

      fdisk -l

      看下我们的设置。

  5. 在分区上创建文件系统 LFS 使用 Linux 内核能识别的任何文件系统，最常用的是 ex3 和 ex4。

    ext2: 适用于分区容量不是太大，更新也不是特别频繁的情况，如 /boot 分区。 ext3: 是 ext2 改进版，支持日志功能。它通常被用作通用的文件系统。 ext4: 是 ext 文件系统的最新版。提供很多新的特性。

    其他的文件系统，包括 FAT32、NTFS 等都在专门领域有独到的作用。

    执行如下的命令 

    mkfs -v -t ext4 /dev/sda1
mkswap /dev/sda2

    现在文件系统已经创建完成了。下一步就是访问这些分区。

  6. 挂载新分区下面我们将新建的分区挂载到选定的挂载点 */mnt/lfs*， 我们这里可以根据自己的喜好更改。 

    export LFS=/mnt/lfs

    下一步，创建挂载点并用下面的命令挂载 LFS 文件系统： 

    mkdir -pv $LFS # 建立挂载点
mount -v -t ext4 /dev/sda1 $LFS # 将/dev/sda1 挂载到 $LFS

    对于交换分区： 

    /sbin/swapon -v /dev/sda2

    补充：什么是挂载？为什么要挂载？

    linux、unix 这类操作系统将系统中的一切都作为文件来管理。在 windows 中我们常见的硬件设备、磁盘分区等，在 linux、unix 中都被视作文件，对设备、 分区的访问就是读写对应的文件。linux 里,没有盘符这个概念,我们用目录的形式进行管理,把每一个分区和某一个目录对应.那么,我们对这个目录的操作就是对这个分区的操作,这样就实现了硬件管理手段和软件目录管理手段的统一,这个把分区和目录对应的过程就叫做挂载,也就是 mount.

    现在准备工作差不多了，我们需要下载软件包。

  7. 软件包和补丁下载的软件包需要保存在某个地方，以方便在整个构建的过程中都能有效访问。 我们可以用 $LFS/sources 保存软件包和补丁，同时作为工作目录。 

    mkdir -v $LFS/sources

    设置目录的写权限和粘滞模式。“粘滞模式”意思是就算有多个用户对某个目录有写权 限，仍然只有该文件的主人能删除一个粘滞目录里的文件。 

    chmod -v a+wt $LFS/sources

    我们首先创建一个 wget-list，也就是下载的链接文件 

    curl -s https://blog.iandreev.com/wp-content/uploads/2013/09/wget-list.txt >> wget-list.txt

    我们可以这样下载 

    wget -i wget-list -P $LFS/sources

    但是我发现非常非常的慢。

    后来找了一个命令 aria2c, 

    aria2c -j5 --input-file=wget-list.txt

    这部分我是在 host 上下载的，然后我通过 scp 命令传到 vm 中。 

    scp -rp <file or dir> root@vm-ip:/mnt/lfs/sources
